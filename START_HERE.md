# Start Here

This page is for you if you are new to programming, switching careers, coming
back after a long break, or trying to help someone else begin.

You do not need to know the right language yet. You do not need a perfect plan.
You need one small path that helps you touch code, get confused, fix something,
and feel the loop of learning.

## The Short Version

Pick one of these and stay with it for two weeks:

- **Gentlest start:** [Scratch](https://scratch.mit.edu/) plus a visual book like
  [Hello Ruby](https://www.helloruby.com/).
- **Practical adult beginner path:** [Python Programming MOOC](https://programming-25.mooc.fi/)
  or [Python for Everybody](https://www.py4e.com/).
- **Website path:** [MDN Learn Web Development](https://developer.mozilla.org/en-US/docs/Learn)
  plus [freeCodeCamp](https://www.freecodecamp.org/learn/).

After two weeks, build something tiny without following a tutorial exactly.

## A Calm First Week

| Day | Do this | Why |
| --- | --- | --- |
| 1 | Make or remix one Scratch project | Learn sequencing, events, loops, and conditions without syntax friction |
| 2 | Read a gentle book chapter or watch one beginner video | Make the ideas feel less abstract |
| 3 | Start one beginner course | Move from ideas to typed code |
| 4 | Type examples yourself | Your hands need practice too |
| 5 | Break a tiny program on purpose and fix it | Debugging is normal |
| 6 | Make a calculator, quiz, or guessing game | Finish something small |
| 7 | Write what confused you and what clicked | Build your own map |

## If You Are a Career Switcher

You do not need to learn every computer science topic before you can make
progress. Your first useful goal is proof: a few small projects you understand
well enough to explain.

Choose one track for 8-12 weeks:

| Track | Learn | Build |
| --- | --- | --- |
| Web developer | HTML, CSS, JavaScript, Git, APIs | personal site, form app, API app, full stack CRUD app |
| Python automation | Python, files, APIs, CSV/Excel, SQL | file organizer, report generator, API client, dashboard |
| Data analyst | SQL, spreadsheets, Python, statistics | cleaned dataset, SQL analysis, notebook, dashboard |
| AI app builder | Python, APIs, prompts, embeddings, evaluation | chatbot, document search, classifier, evaluation set |

Try to make one project solve a real problem from your previous career. That
context is an advantage.

## How to Use AI Without Losing the Learning

AI is part of programming now. It can explain errors, rephrase documentation,
suggest practice problems, and give hints when you are stuck.

Use AI like a tutor, not like a vending machine for answers.

Good prompts:

```text
I am learning Python. Do not give me the full answer yet.
Ask me one question at a time and give hints when I get stuck.
Here is what I tried:
<paste your code>
```

```text
Explain this error message like I am new to programming.
Then give me two things to check before you show me a solution.
```

```text
Review this small project for edge cases.
Do not rewrite it unless I ask.
Tell me what I should test next.
```

Avoid:

- pasting assignments and accepting full answers
- using code you cannot explain
- skipping the official docs entirely
- believing a generated answer without testing it
- confusing "it runs" with "I understand it"

## What Counts as Progress

Progress is not just finishing courses. Progress looks like:

- You can read an error message without panicking.
- You can explain what a variable is.
- You can change code and predict what might happen.
- You can search the docs for one function.
- You can ask a clearer question than you could last week.
- You can finish a tiny project and improve it later.

## Your First Three Projects

Pick three small projects. Keep them boring enough to finish.

1. **Personal page:** who you are, what you are learning, and one thing you made.
2. **Tiny tool:** calculator, unit converter, file renamer, or budget helper.
3. **API or data project:** weather app, recipe search, CSV cleaner, or simple
   dashboard.

Each project should have a short README:

- What does it do?
- How do I run it?
- What did you learn?
- What would you improve next?

## When You Feel Stuck

Being stuck is not a sign that you are bad at this. It is the work.

Try this:

1. Read the error out loud.
2. Check the line number.
3. Undo the last change.
4. Add one print statement or log.
5. Search the exact error.
6. Ask AI for a hint, not the answer.
7. Take a break and come back.

Then write down the fix. Your future self will be grateful.
